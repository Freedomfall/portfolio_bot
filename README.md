# Portfolio Telegram Bot

Мой Telegram-бот для портфолио, написанный на Python и Pyrogram.

Бот показывает информацию обо мне, навыки, проекты и контакты через удобное Telegram-меню.  
Также в проекте реализованы база данных, статистика, админ-панель, обратная связь, антиспам, экспорт данных и production-подход к обработке ошибок.

## Версия

Текущая версия: **1.2.0**

## Команды

### Пользовательские команды

- `/start` — открыть главное меню
- `/menu` — показать главное меню
- `/help` — показать помощь
- `/about_project` — техническое описание проекта
- `/version` — версия бота и список возможностей
- `/roadmap` — планы развития проекта
- `/privacy` — информация о хранении данных
- `/profile` — показать профиль пользователя в боте
- `/myid` — узнать свой Telegram ID
- `/ping` — проверить работу бота и uptime
- `/status` — проверить статус бота
- `/health` — проверить состояние бота и базы данных
- `/coin` — подбросить монетку Орёл или Решка
- `/dice` — бросить кубик от 1 до 6
- `/feedback текст` — отправить сообщение автору бота

### Админ-команды

- `/admin` — открыть админ-панель, только для владельца
- `/stats` — показать статистику бота, только для владельца
- `/admin_logs` — показать последние действия администратора
- `/test_notify` — отправить тестовое уведомление админу
- `/broadcast текст` — отправить сообщение всем пользователям, только для владельца
- `/reply user_id текст` — ответить пользователю, только для владельца
- `/backup_db` — скачать резервную копию базы, только для владельца
- `/last_feedback` — показать последние сообщения обратной связи, только для владельца
- `/export_users` — выгрузить пользователей в CSV, только для владельца
- `/export_feedback` — выгрузить обратную связь в CSV, только для владельца

## Админ-панель

Админ-панель открывается командой:

```text
/admin
```

В ней доступны:

- статистика
- backup базы данных
- Ping / Uptime
- экспорт пользователей
- экспорт feedback
- последние feedback-сообщения
- последние admin logs
- тест уведомления
- инструкция по рассылке
- инструкция по ответу пользователю

## База данных

Бот использует SQLite.

Создаются таблицы:

- `users` — пользователи бота
- `feedback` — сообщения обратной связи
- `admin_logs` — действия администратора

На Railway база хранится через Volume:

```env
DB_PATH=/data/bot_stats.db
```

## Переменные окружения

Файл `.env` должен быть создан локально на основе `.env.example`.

```env
API_ID=your_api_id
API_HASH=your_api_hash
BOT_TOKEN=your_bot_token
ADMIN_ID=your_telegram_id
DB_PATH=/data/bot_stats.db
COIN_GIF_URL=https://example.com/coin.gif
```

`COIN_GIF_URL` необязателен.  
Если переменная не указана или GIF не отправляется, бот использует текстовую анимацию монетки.

## Как запустить проект локально

1. Скачать проект или склонировать репозиторий:

```bash
git clone https://github.com/Freedomfall/portfolio_bot.git
```

2. Перейти в папку проекта:

```bash
cd portfolio_bot
```

3. Создать виртуальное окружение:

```bash
python -m venv venv
```

4. Активировать виртуальное окружение на Windows:

```bash
venv\Scripts\activate
```

5. Установить зависимости:

```bash
pip install -r requirements.txt
```

6. Создать файл `.env` по примеру `.env.example`:

```env
API_ID=your_api_id
API_HASH=your_api_hash
BOT_TOKEN=your_bot_token
ADMIN_ID=your_telegram_id
DB_PATH=bot_stats.db
COIN_GIF_URL=https://example.com/coin.gif
```

7. Запустить бота:

```bash
python main.py
```

## Деплой на Railway

Для Railway нужно добавить переменные в разделе **Variables**:

```env
API_ID=your_api_id
API_HASH=your_api_hash
BOT_TOKEN=your_bot_token
ADMIN_ID=your_telegram_id
DB_PATH=/data/bot_stats.db
COIN_GIF_URL=https://example.com/coin.gif
```

Также нужно подключить Railway Volume и использовать путь:

```env
DB_PATH=/data/bot_stats.db
```

## Проверка кода

Перед коммитом можно проверить синтаксис:

```bash
python -m py_compile main.py
```

Если команда ничего не вывела, значит синтаксических ошибок нет.

```text
.env.example
```
